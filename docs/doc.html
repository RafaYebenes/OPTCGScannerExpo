1. Introducción y Alcance

OPTCG Scanner es una solución móvil desarrollada para facilitar la gestión de inventario de cartas del juego One Piece Card Game. El objetivo principal es reducir la fricción de entrada de datos manual mediante el uso de tecnologías OCR (Reconocimiento Óptico de Caracteres) en tiempo real.

Funcionalidades Principales:

  • Escaneo en tiempo real: Detección automática de códigos de set (ej: OP05-060).
  • Soporte Alt Art / Foil: Lógica para diferenciar cartas normales de variantes raras.
  • Gestión de Inventario: Añadir, eliminar y actualizar cantidades sincronizadas en la nube.
  • Precios de Mercado: Visualización de precios (EUR) obtenidos de Cardmarket (datos estáticos/actualizables).

2. Instalación y Configuración

Requisitos Previos

  • Node.js (v16 o superior)
  • npm o yarn
  • Dispositivo físico (Android/iOS) o Emulador con soporte de cámara.
  • Cuenta en Supabase.

Pasos de Instalación

# 1. Clonar el repositorio
git clone https://github.com/tu-usuario/optcg-scanner.git
cd optcg-scanner

# 2. Instalar dependencias
npm install

# 3. Configurar variables de entorno
# Crear archivo .env en la raíz
cp .env.example .env

Configuración de Entorno (.env)

El proyecto requiere las siguientes claves para conectar con Supabase:

EXPO_PUBLIC_SUPABASE_URL=https://tu-proyecto.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=tu-anon-key-larga-y-secreta

Ejecución

# Iniciar servidor de desarrollo
npx expo start

# Para limpiar caché si hay errores
npx expo start -c

3. Arquitectura del Sistema

El proyecto utiliza una arquitectura basada en React Context para el estado global y una separación estricta entre UI y Lógica.

Capa Descripción Archivos Clave
UI (Vistas) Componentes visuales, pantallas y navegación. ScannerScreen.tsx, CollectionScreen.tsx
Lógica (Hooks) Lógica de negocio reutilizable (ej: procesar OCR). useCardScanner.ts, useCardStorage.ts
Estado (Context) Gestión de datos globales (Usuario, Colección). CollectionContext.tsx, AuthContext.tsx
Servicio Comunicación asíncrona con el Backend. supabaseService.ts

4. Base de Datos (Diccionario de Datos)

A continuación se detallan las sentencias DDL para recrear el esquema en PostgreSQL (Supabase).

Tabla: cards (Catálogo Maestro)

create table public.cards (
  id uuid default gen_random_uuid() primary key,
  code text not null unique,  -- Índice: OP01-001
  name text not null,
  set_code text,              -- OP01, ST01
  rarity text,                -- SR, L, R
  color text,                 -- Red, Blue
  image_url text,
  market_price_eur numeric default 0,
  updated_at timestamp with time zone default now()
);

create index idx_cards_code on public.cards(code);

Tabla: user_collection (Inventario)

create table public.user_collection (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users not null,
  card_id uuid references public.cards(id) not null,
  quantity integer default 1,
  is_foil boolean default false,  -- Flag para AA/Foil
  scanned_at timestamp with time zone default now(),
  
  -- Restricción: Un usuario no debería tener filas duplicadas 
  -- para la misma carta y estado foil (opcional)
  unique(user_id, card_id, is_foil)
);

5. Lógica del Negocio (Core)

5.1 Algoritmo de Parseo (OCR)

El archivo utils/cardCodeParser.ts contiene la lógica para extraer códigos válidos del "ruido" que genera el OCR.

Regex Pattern: /(OP|ST|EB|P)\d{2}-\d{3}/i

El flujo de limpieza es:

  1. Normalización: Reemplazar ceros por 'O' en prefijos (ej: 0P01 -> OP01).
  2. Filtrado: Descartar textos de menos de 5 caracteres.
  3. Validación: Comprobar si cumple el formato XX00-000.
  4. Debounce: Si el código es igual al último escaneado hace menos de 2 segundos, se ignora.

5.2 Lógica de Variantes (Alt Art)

Cuando el usuario activa el modo "AA" en ScannerScreen:

  1. El estado isAltMode pasa a true.
  2. Se envía este booleano a supabaseService.addCardToCollection.
  3. El servicio busca en la tabla cards si existe una variante con sufijo _p1, _p2, etc.
  4. Si existe la variante en DB, se usa ese card_id. Si no, se usa el ID de la carta base pero se marca is_foil = true en la colección del usuario.

6. Servicios y API

El archivo supabaseService.ts actúa como la única puerta de enlace a los datos.

Método Parámetros Retorno Descripción
addCardToCollection userId, code, isFoil { success, data } Busca la carta por código, resuelve variantes AA e inserta/actualiza.
getUserCollection - CollectionItem[] Obtiene todas las cartas del usuario actual con un JOIN a la tabla cards.
updateCardQuantity id, quantity boolean Actualiza cantidad. Si es <= 0, elimina la fila.

7. Solución de Problemas Comunes

Error: "Carta no encontrada en base de datos maestra"

Causa: El código escaneado (ej: OP09-001) pertenece a un set nuevo que aún no ha sido cargado en la tabla cards.

Solución: Ejecutar el script json_to_sql.py con el archivo JSON más reciente para actualizar el catálogo.

Error: "Network Request Failed" / Conexión DB

Causa: Supabase ha bloqueado conexiones IPv4 directas en el plan gratuito.

Solución: Utilizar el Connection Pooler (Puerto 6543) en lugar del puerto directo (5432) si se conecta desde DBeaver. O usar la CLI de Supabase.

La cámara no enfoca cartas brillantes (Foil)

Causa: Reflejos en la superficie de la carta confunden al OCR.

Solución: Se implementó un control de Zoom (1.5x por defecto) y un botón de Flash manual para mitigar sombras y reflejos.
